Spring PetClinic on Red Hat UBI9 / JDK 21 under the same HTTP load as `samples/petclinic`, recorded inside a 1 CPU / 2 GiB container. Same before/after matrix as `samples/petclinic-ubi8` — legacy ParallelGC tuning (the classic OpenShift `run-java.sh` defaults) vs G1GC with equivalent, G1-appropriate settings — but on the newer UBI9 base image and JDK 21.

```
samples/petclinic-ubi9/
├── before/                ParallelGC + adaptive-sizing knobs
│   ├── Dockerfile         registry.access.redhat.com/ubi9/openjdk-21
│   ├── docker-compose.yml recording harness (1 CPU / 2 GiB, 120 s JFR)
│   ├── petclinic.jfr      (gitignored, regenerated by docker compose up)
│   ├── report.md          jfrdoc report (generated)
│   └── *.json             raw tool outputs (generated)
└── after/                 G1GC with G1-specific equivalents
    ├── Dockerfile         same UBI9 / JDK 21 image
    ├── docker-compose.yml recording harness
    ├── petclinic.jfr      (gitignored)
    ├── report.md
    └── *.json
```

## JVM flags

Identical to `samples/petclinic-ubi8` so the two samples isolate the base-image/JDK change. Both scenarios share `-Xms342m -Xmx1367m -XX:MaxMetaspaceSize=200m -XX:MinHeapFreeRatio=10 -XX:MaxHeapFreeRatio=70` and the 1 CPU / 2 GiB container limits.

| | before (ParallelGC) | after (G1GC) |
|---|---|---|
| Collector | `-XX:+UseParallelGC` | `-XX:+UseG1GC` |
| Goal | `-XX:GCTimeRatio=4` (throughput) | `-XX:MaxGCPauseMillis=200` (pause) |
| Generation sizing | `-XX:NewRatio=5`, `-XX:AdaptiveSizePolicyWeight=90` | dropped — G1 sizes the young gen adaptively; pinning it via `NewRatio` disables that |
| Concurrent cycle | n/a | `-XX:InitiatingHeapOccupancyPercent=35` |
| Extras | — | `-XX:+UseStringDeduplication` (G1-only) |

Unlike the JDK 17 sample, JDK 21 emits `jdk.NativeMemoryUsage` JFR events (added in JDK 20), so `-XX:NativeMemoryTracking=summary` here produces real NMT data in the recording and jfrdoc can do full container-fit analysis.

## Refresh the snapshots

Two independent steps, same as `samples/petclinic`. The snippets show the `before` half; swap `before` for `after` everywhere for the other half.

**Step 1 — record the JFR** (needs `docker` + `docker compose`):

```bash
docker compose -f samples/petclinic-ubi9/before/docker-compose.yml up --build --abort-on-container-exit --exit-code-from loadgen
docker compose -f samples/petclinic-ubi9/before/docker-compose.yml down
```

Each compose file bind-mounts its own directory as `/jfr`, so the recording lands at `samples/petclinic-ubi9/before/petclinic.jfr` the moment the JVM finalizes it.

**Step 2 — analyze with jfrdoc** (needs Java 25 + your Anthropic key):

```bash
JFR=samples/petclinic-ubi9/before/petclinic.jfr
OUT=samples/petclinic-ubi9/before

./jfrdoc analyze "$JFR" \
    --framework spring --container-cpu 1 --container-memory 2Gi \
    > "$OUT/report.md"
```
