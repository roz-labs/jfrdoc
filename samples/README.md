Sample `.jfr` recordings for testing go here. JFR files themselves are not
committed (see `.gitignore`); generated reports and tool-output JSON are.

## Synthetic micro-recording

Tiny 8-second allocation/CPU workload, handy for fast local iteration:

```
./gen-sample.sh        # writes samples/sample.jfr
```

Requires Java 25+ on PATH.

## Real-application case study

Spring PetClinic under HTTP load, recorded inside a 1 CPU / 2 GiB container.

```
samples/petclinic/
├── case-study.md          before/after walkthrough
├── before/                fat-jar `java -jar app.jar`
│   ├── petclinic.jfr      (gitignored, regenerated by docker compose up)
│   ├── report.md          jfrdoc report
│   └── *.json             raw tool outputs (handy as prompt-tuning fixtures)
└── after/                 exploded layout + -XX:+UseG1GC -Xmx1g
    ├── petclinic.jfr      (gitignored)
    ├── report.md
    └── *.json
```

See [`petclinic/case-study.md`](petclinic/case-study.md) for the reproduce
commands and the side-by-side metric diff.
